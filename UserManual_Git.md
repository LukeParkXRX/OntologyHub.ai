# 깃허브(GitHub) & 터미널 사용 설명서

## 1. 터미널이 말을 안 들을 때 (입력이 안 될 때)
스크린샷을 보니 **서버가 실행 중인 화면**(`npm run dev` 등)에서 명령어를 입력하셨군요!
서버가 켜져 있으면 터미널은 계속 그 일을 하느라 바빠서 사용자 명령을 듣지 못합니다.

### 해결 방법
1. **새 터미널 열기**: 터미널 창 우측 상단의 `+` 버튼을 눌러 **새로운 터미널 창**을 띄우세요. 그곳에서 명령어를 입력하면 됩니다.
2. **서버 끄기**: 만약 서버를 끄고 싶다면, 서버가 돌아가는 터미널에서 `Ctrl + C` 키를 누르면 종료됩니다.

---

## 2. 회사 -> 집 (장소 이동 시 최신 파일 받기)
회사에서 작업을 마치고 깃허브에 올린(`Push`) 후, 집에 도착해서 다시 작업을 시작하기 전에 **반드시** 아래 명령어로 최신 파일을 받아와야 합니다.

### 명령어: `git pull`
```bash
git pull origin main
```
> **의미**: "깃허브(Origin)의 메인(Main) 저장소에서 최신 파일을 당겨와줘(Pull)"

### 작업 순서 루틴
1. **[집] 작업 시작 전**: `git pull origin main` (최신파일 받기)
2. **[집] 작업 후**: `git add .` -> `git commit -m "내용"` -> `git push origin main` (저장해서 올리기)
3. **[회사] 출근 후**: `git pull origin main` (집에서 한 거 받기)

---

## 3. 요약
- **서버 켜져 있음** -> `+` 눌러서 새 창 띄우기
- **작업 시작 전** -> `git pull origin main`
- **작업 끝난 후** -> `git add` + `git commit` + `git push`
